Bases: object

Class responsible for performing the MCMC optimization.

Contains functionality for initialize MCMC runs and different methods of randomly moving in the order space until convergence (or iteration limit)

Parameters

  • initial_order (Order) – Order

  • search_space (SearchSpace object) – Search space

  • score_calculator (ScoreCalculator object) – An object which calculates the score for given order

  • greedy (boolean) – Whether to compute a greedy permutation in the reordering step (NOT YET IMPLEMENTED)

  • MAP (boolean) – Whether to use the MAP (Maximum a Posteriori) score to represent an order

  • annealing (bool) – Flag indicating whether simulated annealing should be used to reduce the

temperature as the run continues (score becomes more relevant as the run continues) :type annealing: boolean :type use_nbd: bool :param use_nbd: Flag determining whether to use steps which only consider adjacent nodes :type use_nbd: boolean :type verbosity: Literal[0, 1, 2] :param verbosity: The level of verbosity. 0 = silent, 1 = progress bar + intermittent logging, 2 = progress bar + detailed per step logging :type verbosity: VerbosityLevel

Perform a step.
