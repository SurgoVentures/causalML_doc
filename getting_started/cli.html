The easiest way to run intervene is via the text-based command-line interface (CLI).

One hyphen can be used if an option has a short-form (-d for â€“data, for instance) but not all commands have this option. A summary of the options, along with their default values, can be displayed by typing the command followed by -h or â€“help:

One type of option that warrants special mention is the â€˜switch-optionâ€™. Options of this type are an on/off toggle, and do not need to be parameterised. One example is the option â€“no-sl, belonging to the â€˜realâ€™ subcommand: there is no need to follow the option with True; typing the option alone will disable the structure learning component.

The main subcommands in intervene are:

generating and running synthetic data.

real

running real data.

preprocessing real data.

Detailed guides on the specific subcommands can be found in the subsequent sections of this guide.

LimitationsÂ¶

The CLI has been designed to cover most use-cases but the options are by no means sufficient to cover every element of intervene as to do so would be overwhelming considering the range of possibilities. There may be components in intervene that are absent from the predefined pipelines in which case a custom pipeline needs to be written (see section Advanced Usage).
